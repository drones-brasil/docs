## 1. O que é uma FSM
Uma FSM (Finite State Machine) descreve sistemas que:
- estão em **um estado** por vez
- mudam por **transições**
- acionadas por **eventos/condições**

## 2. Estado: `on_enter`, `act`, `on_exit`
Cada estado costuma ter:
- `on_enter`: inicialização
- `act`: loop principal do estado
- `on_exit`: limpeza/finalização

## 3. Blackboard
O blackboard é um “quadro negro” compartilhado entre estados para dados e ponteiros.

Exemplo:

```cpp
// Set
blackboard.set<float>("takeoff_height", -2.0);

// Get
float h = *blackboard.get<float>("takeoff_height");
```

## 6. Implementando Takeoff & Landing (exemplo completo)
O padrão é:
- FSM com transições
- estados implementando `act` e retornando strings de eventos
- nó ROS 2 que chama `execute()` num timer

## 7. Implementando GoTo
Extensão natural de takeoff/landing:
- adiciona `GoToState`
- define `target_base` no blackboard
- transições TAKEOFF → GOTO → LANDING
